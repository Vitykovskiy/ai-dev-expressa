# Сценарий: administrator блокирует пользователя

## Граница

Один сценарий: лишение пользователя доступа к продукту.

## Параметры сценария

- Триггер: administrator принимает решение запретить пользователю использование продукта.
- Актор: administrator.
- Цель: прекратить доступ пользователя к приложению.
- Исход: пользователь заблокирован и не может пользоваться приложением.

## Основной поток

1. Administrator открывает backoffice через Telegram backoffice-бота.
2. Administrator переходит во вкладку `Пользователи`.
3. Administrator выбирает пользователя.
4. Administrator выполняет блокировку пользователя.
5. Система сохраняет статус блокировки.
6. Пользователь теряет доступ к приложению.

## Альтернативы и исключения

- Отсутствуют в исходных требованиях.

## Связанные правила

- `AR-004`
- `BO-006`, `BO-013`
