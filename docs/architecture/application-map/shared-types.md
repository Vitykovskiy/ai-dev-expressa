# Карта общих типов

## Назначение

Эта карта фиксирует фактическое устройство `packages/shared-types`: какие общие контракты опубликованы для нескольких контуров и когда задачи должны обновлять общий пакет. Её читают только задачи, меняющие DTO, перечисления, причины отказа или другие общие контракты.

## Ответственность и зависимости

| Контур | Путь | Ответственность | Основные зависимости |
| --- | --- | --- | --- |
| Общие типы | `packages/shared-types` | Общий пакет деклараций типов для контрактов между `apps/server` и веб-приложениями | `apps/server`, `apps/backoffice-web`, будущие веб-приложения |

## Текущий состав

- Типы bootstrap доступа во внутренний административный контур.
- DTO запроса и ответа для серверного bootstrap доступа.
- Роли, причины отказа и список допустимых вкладок внутреннего административного контура.
- DTO структурного снимка каталога меню для `FEATURE-002`.
- Классификаторы товара, размеры напитков, режим выбора группы дополнительных опций и коды бизнес-ошибок сохранения каталога меню.

## Правила использования

- Публичный контракт между серверной и клиентской частью сначала фиксируется в `packages/shared-types`, затем реализуется в соответствующих контурах.
- Серверная и клиентская часть не должны создавать локальные дубли общих DTO.
- Минимальный набор бизнес-ошибок `FEATURE-002` общий для серверной и клиентской части: `administrator-role-required`, `invalid-drink-size-model`, `invalid-option-group-rule`.
- Любое расширение перечня общих ошибок, ролей, вкладок или DTO сначала фиксируется в общем пакете.

## Команды проверки

- Установка зависимостей: `npm install`.
- Проверка потребителей общего пакета выполняется через сборку и тесты затронутых контуров: `apps/server`, `apps/backoffice-web` или будущих веб-приложений.

## Когда обновлять карту

- Добавлен или изменён общий DTO.
- Изменился перечень ролей, вкладок, причин отказа или бизнес-ошибок.
- Появился новый потребитель `packages/shared-types`.
- Изменился порядок публикации или проверки общего пакета.
