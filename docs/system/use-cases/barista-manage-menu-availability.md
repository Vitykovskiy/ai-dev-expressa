# Вариант использования: Barista Manage Menu Availability

## Граница

Один вариант использования: временное изменение barista доступности товара или дополнительной опции.

## Источники

- `docs/business/scenarios/barista-manage-menu-availability.md`
- `docs/business/business-rules/backoffice-operations.md`
- `docs/business/business-rules/menu-catalog-and-options.md`

## Триггер

Barista инициирует изменение доступности товара или дополнительной опции.

## Предусловия

- Пользователь идентифицирован через служебного Telegram-бота.
- Пользователь имеет доступ к вкладке `Доступность`.

## Основной поток

1. Система предоставляет barista перечень товаров и дополнительных опций, доступных для оперативного управления.
2. Barista выбирает товар или дополнительную опцию.
3. Barista изменяет признак доступности.
4. Система сохраняет новое значение доступности.
5. Система применяет новое значение к customer-сценарию выбора товаров.

## Альтернативы и исключения

- Если barista пытается изменить цену, структуру меню или состав групп опций, система не допускает такое действие в этом варианте использования.

## Постусловия

- Для выбранного товара или дополнительной опции обновлена временная доступность.
