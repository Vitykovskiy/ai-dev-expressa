# Domain Model: Menu Catalog

## Граница

Одна доменная модель: каталог меню, товары, размеры напитков, группы дополнительных опций и управляемая доступность для `Expressa v1`.

## Сущности и value objects

### Entity `MenuCategory`

| Поле | Тип семантики | Описание |
|---|---|---|
| `menuCategoryId` | Идентификатор | Идентификатор категории меню. |
| `name` | Атрибут | Наименование категории. |
| `optionGroupRefs` | Связь | Группы дополнительных опций, назначенные на категорию. |

### Entity `MenuItem`

| Поле | Тип семантики | Описание |
|---|---|---|
| `menuItemId` | Идентификатор | Идентификатор товара. |
| `menuCategoryId` | Связь | Категория, к которой относится товар. |
| `name` | Атрибут | Наименование товара. |
| `basePrice` | Денежное значение | Цена товара или базовая цена в зависимости от типа товара. |
| `availability` | Признак состояния | Временная доступность товара для customer-сценария. |
| `itemType` | Классификатор | Отличает напиток от товара без обязательного выбора размера. |

### Value object `DrinkSize`

Допустимые значения:

- `S`
- `M`
- `L`

### Entity `DrinkSizePrice`

| Поле | Тип семантики | Описание |
|---|---|---|
| `menuItemId` | Связь | Напиток, к которому относится размер. |
| `size` | Значение размера | Один из размеров `S`, `M`, `L`. |
| `price` | Денежное значение | Цена напитка для выбранного размера. |

### Entity `OptionGroup`

| Поле | Тип семантики | Описание |
|---|---|---|
| `optionGroupId` | Идентификатор | Идентификатор группы дополнительных опций. |
| `name` | Атрибут | Наименование группы. |
| `selectionMode` | Правило выбора | Фиксирует, допускается множественный выбор или только один вариант. |
| `availability` | Признак состояния | Временная доступность группы для customer-сценария не зафиксирована во входных материалах. |

### Entity `Option`

| Поле | Тип семантики | Описание |
|---|---|---|
| `optionId` | Идентификатор | Идентификатор дополнительной опции. |
| `optionGroupId` | Связь | Группа, в которую входит опция. |
| `name` | Атрибут | Наименование опции. |
| `priceDelta` | Денежное значение | Доплата за опцию; может быть нулевой. |
| `availability` | Признак состояния | Временная доступность опции для customer-сценария. |

## Отношения

- `MenuCategory` содержит множество `MenuItem`.
- `MenuCategory` наследует на свои товары множество `OptionGroup`.
- `OptionGroup` содержит множество `Option`.
- `MenuItem` типа `drink` должен иметь набор цен `DrinkSizePrice`.

## Инварианты и ограничения

- Меню отображается customer по категориям.
- Для товара customer может выбрать количество и доступные дополнительные опции до добавления в корзину.
- Для напитка выбор размера обязателен.
- Цена напитка зависит от выбранного размера.
- Дополнительные опции не отображаются как самостоятельные позиции меню.
- Дополнительная опция может быть платной или бесплатной.
- Если группа не взаимоисключающая, customer может выбрать несколько опций из группы.
- Если группа взаимоисключающая, customer может выбрать не более одной опции из группы.
- `barista` может менять только доступность товара и дополнительной опции.
- `barista` не может изменять структуру меню и цены.
- `administrator` может изменять структуру меню, товары, цены и группы дополнительных опций.

## Наблюдаемые пробелы

- Во входных материалах не зафиксировано, можно ли временно менять доступность всей группы дополнительных опций как отдельной сущности; зафиксировано только изменение доступности позиций меню и дополнительных опций.
- Не определено, как ведёт себя заказной состав при последующем изменении цены товара или опции; следовательно, необходимый уровень snapshot-данных в заказе должен быть зафиксирован отдельным системным решением.
