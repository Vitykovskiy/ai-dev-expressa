# Сценарий: barista получает напоминание о заказах, ожидающих действий

## Граница

Один сценарий: напоминание barista о необходимости обработать заказ.

## Параметры сценария

- Триггер: в системе есть заказы, ожидающие действий barista.
- Актор: barista.
- Цель: не пропустить заказ, требующий обработки.
- Исход: barista получает Telegram-напоминание через служебного Telegram-бота.

## Основной поток

1. В системе накапливаются заказы, ожидающие действий barista.
2. Система формирует Telegram-напоминание.
3. Backoffice-бот отправляет напоминание barista.
4. Barista возвращается к обработке заказов в backoffice.

## Альтернативы и исключения

- Точная периодичность напоминаний не определена в исходных требованиях.
