# Use Case: Administrator Manage Slot Settings

## Граница

Один use case: изменение administrator рабочих часов и вместимости слотов выдачи.

## Источники

- `docs/business/scenarios/administrator-manage-slot-settings.md`
- `docs/business/business-rules/pickup-slots-and-capacity.md`
- `docs/business/business-rules/backoffice-operations.md`

## Триггер

Administrator инициирует изменение параметров выдачи.

## Предусловия

- Пользователь идентифицирован через Telegram backoffice-бота.
- Пользователь имеет роль `administrator` и доступ к вкладке `Настройки`.

## Основной поток

1. Система предоставляет administrator текущие рабочие часы и текущую вместимость слотов.
2. Administrator изменяет рабочие часы и/или вместимость слотов.
3. Система сохраняет новые значения.
4. Система применяет новые параметры к формированию доступных слотов выдачи.

## Альтернативы и исключения

- Если значения не изменялись, продолжают действовать текущие настройки, по умолчанию `09:00–20:00` и `5`.

## Постусловия

- Новые рабочие часы и/или вместимость используются в дальнейших операциях выбора слотов.
