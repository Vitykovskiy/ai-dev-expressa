# Карта внутреннего административного контура

## Назначение

Эта карта фиксирует фактическое устройство `apps/backoffice-web`: ответственность клиентской части, маршруты, хранилища состояния, HTTP-клиенты, зависимости, команды запуска и feature-срезы. Её читают только задачи, которые меняют Vue-приложение внутреннего административного контура.

## Ответственность и зависимости

| Контур | Путь | Ответственность | Основные зависимости |
| --- | --- | --- | --- |
| Внутренний административный контур | `apps/backoffice-web` | Vue-приложение для UI ролей `barista` и `administrator`: root layout, bootstrap доступа, восстановление сессии, серверно-управляемая навигация вкладок, route guards и клиентский слой управления каталогом меню | `Vue 3`, `Vite`, `Vuetify`, `Vue Router`, `Vitest`, `packages/shared-types` |

## Модули и точки входа

- `apps/backoffice-web/src/main.ts` — точка входа клиентской части.
- `apps/backoffice-web/src/App.vue` — корневой компонент.
- `apps/backoffice-web/src/router` — маршруты, route guards и вложенная навигация вкладки `menu`.
- `apps/backoffice-web/src/layouts` — каркас внутреннего административного контура.
- `apps/backoffice-web/src/pages` — страницы вкладок и подпотоков.
- `apps/backoffice-web/src/services` — HTTP-клиенты, окружение, Telegram WebApp-адаптер и хранение `accessToken`.
- `apps/backoffice-web/src/stores` — хранилища состояния доступа и каталога меню.
- `apps/backoffice-web/src/composables` — композиционные функции для состояния layout и редакторов.
- `apps/backoffice-web/src/components` — переиспользуемые UI-компоненты текущих вкладок.
- `apps/backoffice-web/.env.example` — пример переменных окружения клиентской части.
- Пакет `packages/ui` не планируется: клиентское веб-приложение и внутренний административный контур развиваются независимо в своих приложениях.

## Feature-срезы

### `FEATURE-001`: доступ во внутренний административный контур

- Приложение читает Telegram-контекст из веб-приложения и выполняет pre-bootstrap проверку канала входа.
- При `VITE_DISABLE_TG_AUTH=true` test environment переключается на test-mode payload c `VITE_TEST_TELEGRAM_ID`.
- Для постоянного `test`-окружения VPS `VITE_TEST_TELEGRAM_ID` публикуется build-time конфигурацией, чтобы `QA` мог запускать browser-only e2e без Telegram.
- HTTP bootstrap доступа выполняется через серверный контракт `POST /api/backoffice/access/bootstrap`.
- Сессия восстанавливается через `GET /api/backoffice/access/me` по сохранённому `accessToken`.
- Пользовательский контекст хранится на клиентской стороне, но права доступа не вычисляются локально.
- Навигация вкладок строится из `availableTabs`, полученных от серверной части.
- Route guards останавливают прямой переход к недоступным вкладкам и показывают экран отказа в доступе.
- Для локального каркаса базовые переменные окружения лежат в `apps/backoffice-web/.env.example`; на этапе `FE-001` обязательны `VITE_APP_TITLE` и `VITE_API_BASE_URL`.
- Для локального и VPS test environment дополнительно фиксируются `VITE_DISABLE_TG_AUTH` и `VITE_TEST_TELEGRAM_ID`; эти переменные не заменяют серверные guard-правила и не дают доступ без серверного решения.

### `FEATURE-002`: управление каталогом меню

- `src/services/backoffice-menu-catalog-api.ts` читает и сохраняет структурный снимок каталога меню через существующий серверный API.
- `src/stores/menu-catalog-store.ts` хранит единый черновик вкладки `menu`, dirty-флаг и операции над структурным снимком.
- `src/router/menu-catalog-navigation.ts` содержит вложенные маршруты `menu.menu_categories`, `menu.menu_products`, `menu.menu_product_detail`, `menu.addon_group_detail`.
- Страницы `src/pages/Menu*.vue` покрывают подпотоки категорий, товаров, карточки товара и карточки группы дополнительных опций.
- `src/components/MenuCategory*.vue`, `MenuProductEditorForm.vue`, `MenuAddonGroupEditorForm.vue` и `MenuCatalogSavePanel.vue` реализуют текущие компоненты редакторов категорий, товаров, групп дополнительных опций и сохранения.
- `src/composables/menu-category-editor.ts`, `menu-product-editor.ts` и `menu-addon-group-editor.ts` содержат UX-валидации редакторов категорий, товаров, цен и групп дополнительных опций.
- Клиентская часть может выполнять только UX-валидации обязательности полей и целостности формы; серверная часть остаётся источником истины по модели каталога.
- `FE-004` отвечает за чтение снимка, навигацию подпотоков и базовую оркестрацию состояния.
- `FE-005` добавляет общий черновик снимка, dirty-флаг, сохранение через общий shell вкладки `menu`, отображение ошибок серверного сохранения, редактор категорий и UX-валидацию обязательного наименования.
- `FE-006` добавляет редактор товаров и цен: создание через маршрут `menu.menu_product_detail` с `productId=new`, изменение существующей позиции, переключение `product`/`drink`, базовую цену товара и цены S/M/L для напитка.
- `FE-007` добавляет редактор групп дополнительных опций: создание через маршрут `menu.addon_group_detail` с `optionGroupId=new`, изменение названия, режима `single`/`multiple`, вариантов с `priceDelta` и назначений на категории через `optionGroupRefs` без дублирования серверных правил.

## Маршруты вкладки `menu`

- Верхнеуровневая вкладка `menu` остаётся единственной точкой входа в административное управление каталогом.
- Путь `/menu` редиректится на `/menu/categories` на родительском маршруте вкладки без отдельного пустого дочернего маршрута.
- Подпути вкладки: `/menu/categories`, `/menu/categories/:categoryId/products`, `/menu/categories/:categoryId/products/:productId`, `/menu/categories/:categoryId/addon-groups/:optionGroupId`; создание товаров и групп дополнительных опций использует технический идентификатор `new` в последнем сегменте соответствующего маршрута.
- Все подпути остаются внутри существующего `administrator`-guard.

## Команды запуска и проверки

- Установка зависимостей: `npm install`.
- Локальный запуск клиентской части: `npm run dev --workspace @expressa/backoffice-web`.
- Сборка клиентской части: `npm run build --workspace @expressa/backoffice-web`.
- Модульные тесты клиентской части: `npm run test --workspace @expressa/backoffice-web`.
- Сквозная проверка с серверной частью: `npm run test:e2e`.
- Сквозная проверка против развёрнутого `test`-окружения VPS: `E2E_BASE_URL=<BACKOFFICE_PUBLIC_URL> npm run test:e2e`.
- На этапах `FE-004`–`FE-007` модульные тесты дополнительно покрывают HTTP-клиент каталога меню, хранилище состояния вкладки `menu`, router-переходы, layout-обработку вложенных маршрутов `menu.*`, операции черновика категорий, товаров и групп дополнительных опций, UX-валидации редакторов категорий, товаров, цен и групп дополнительных опций, а также smoke-сценарии создания или изменения категории, товара и группы дополнительных опций.

## Когда обновлять карту

- Изменилась структура `apps/backoffice-web`, точка входа, маршрут или хранилище состояния.
- Изменился HTTP-клиент, shared DTO, guard-поведение или feature-срез клиентской части.
- Появилась новая клиентская зависимость или переменная окружения.
- Изменился способ запуска, сборки или тестирования клиентской части.
