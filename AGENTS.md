# AGENTS

## Старт работы

Перед началом любой работы в этом репозитории агент должен сначала прочитать [README.md](./README.md).

## Зачем это нужно

- `README.md` содержит актуальную структуру проекта.
- `README.md` указывает, где находятся основные документы, шаблоны и задачи.
- `README.md` задает базовую точку входа для навигации по репозиторию.
- `terms-map.md` фиксирует рекомендуемую терминологию для архитектурной и проектной документации.

## Обязательное правило

Если агент еще не прочитал `README.md`, он не должен переходить к анализу, изменению файлов или постановке задач.

При подготовке и обновлении документации агент должен использовать терминологию из [terms-map.md](./terms-map.md) и придерживаться рекомендуемых русских аналогов, если для термина там задана замена.

После внесения любых изменений агент должен проверить, что [README.md](./README.md) остается актуальным, и при необходимости сразу обновить его.

При создании коммитов агент должен использовать формат Conventional Commits.

## Планирование больших задач

Если задача выглядит большой, агент не должен выполнять ее монолитно. К признакам большой задачи относятся много шагов, широкий набор файлов, несколько подсистем, длинный обязательный набор материалов для чтения и необходимость декомпозиции на независимые части.

Перед активным выполнением такой задачи агент обязан составить краткий пошаговый план с порядком выполнения, зависимостями и критериями завершения.

После этого агент обязан декомпозировать работу на независимые подзадачи с минимальным пересечением контекста и зон ответственности.

Независимые подзадачи должны выполняться через субагентов. Основной агент остается координатором, следит за целостностью результата, объединяет вывод и отвечает за финальную интеграцию.

Нельзя назначать двум субагентам пересекающиеся зоны правки или взаимозависимые подзадачи без заранее определенного порядка координации.
