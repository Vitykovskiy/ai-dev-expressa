# Вариант использования: Barista Reject Order

## Граница

Один вариант использования: отклонение barista заказа с обязательной причиной.

## Источники

- `docs/business/scenarios/barista-reject-order.md`
- `docs/business/business-rules/backoffice-operations.md`
- `docs/business/business-rules/order-lifecycle-and-history.md`
- `docs/business/business-rules/pickup-slots-and-capacity.md`

## Триггер

Barista открывает заказ из очереди и инициирует отклонение.

## Предусловия

- Пользователь идентифицирован через служебного Telegram-бота.
- Пользователь имеет роль `barista` или роль с доступом к вкладке `Заказы`.
- Заказ находится в статусе `Создан`.

## Основной поток

1. Система предоставляет barista детали заказа.
2. Barista вводит причину отклонения.
3. Barista инициирует отклонение заказа.
4. Система проверяет наличие причины отклонения.
5. Система проверяет допустимость перехода `Создан` -> `Отклонен`.
6. Система переводит заказ в статус `Отклонен`.
7. Система сохраняет причину отклонения.
8. Система сохраняет идентификатор barista, отклонившего заказ.
9. Система освобождает вместимость выбранного слота.
10. Система отправляет customer уведомление об отклонении с причиной.

## Альтернативы и исключения

- Если причина отклонения не указана, система не выполняет отклонение.
- Если заказ уже вышел из статуса `Создан`, система не выполняет отклонение.

## Постусловия

- Заказ имеет статус `Отклонен`.
- Причина отклонения и audit по отклонению сохранены.
- Вместимость слота освобождена.
