# AGENTS

## Старт работы

Перед началом любой работы в этом репозитории агент должен сначала прочитать [README.md](./README.md).

## Зачем это нужно

- `README.md` содержит актуальную структуру проекта.
- `README.md` указывает, где находятся основные документы, шаблоны и задачи.
- `README.md` задает базовую точку входа для навигации по репозиторию.
- `terms-map.md` фиксирует рекомендуемую терминологию для архитектурной и проектной документации.

## Обязательное правило

Если агент еще не прочитал `README.md`, он не должен переходить к анализу, изменению файлов или постановке задач.

При подготовке и обновлении документации агент должен использовать терминологию из [terms-map.md](./terms-map.md) и придерживаться рекомендуемых русских аналогов, если для термина там задана замена.

После внесения любых изменений агент должен проверить, что [README.md](./README.md) остается актуальным, и при необходимости сразу обновить его.

При создании коммитов агент должен использовать формат Conventional Commits.

## Обязательная подготовка контекста для задач

Если агент получает в работу любую карточку из каталога `tasks/`, он не должен сразу выполнять реализацию этой карточки.

Сначала агент обязан подготовить контекстный пакет: общий план выполнения и набор файлов контекста для подзадач.

Пользователь не обязан отдельно просить о подготовке контекста. Обычная формулировка "возьми задачу в работу" для карточки из `tasks/` означает сначала подготовку контекста, а не немедленную реализацию.

Исключение допускается только для мелких запросов вне карточек `tasks/`: ответы на вопросы, точечный поиск, простые проверки, review без реализации или явно одношаговые правки.

## Роль сборщика контекста

Сборщик контекста — это субагент, который собирает самодостаточный рабочий контекст по одной подзадаче.

Сборщик контекста читает назначенные документы и файлы, фиксирует факты, ограничения, зависимости, границы чтения и правок, проверки и риски.

Сборщик контекста не выполняет исходную задачу, не меняет продуктовый код и не редактирует рабочие артефакты реализации.

Если среда поддерживает субагентов, основной агент обязан использовать субагентов-сборщиков контекста для подготовки контекста по карточкам из `tasks/`.

## Процесс подготовки контекста

Основной агент остается координатором подготовки контекста.

Перед декомпозицией основной агент читает `README.md`, `terms-map.md`, исходную карточку из `tasks/` и обязательные документы, указанные в карточке.

Основной агент декомпозирует задачу на независимые подзадачи с минимальным пересечением зон ответственности и правок.

Каждому субагенту назначается один непересекающийся срез: например один слой клиентской части, один набор компонентов, один контур проверок или один документационный срез.

Нельзя назначать двум субагентам пересекающиеся зоны правок или взаимозависимые подзадачи без заранее определенного порядка координации.

Если задача настолько мала, что декомпозиция невозможна, основной агент все равно создает один контекстный пакет и явно фиксирует, что задача выполняется одной подзадачей.

Основной агент объединяет результаты субагентов, устраняет дубли, фиксирует зависимости и отвечает за целостность итогового плана.

На этапе подготовки контекста основной агент не выполняет исходную задачу.

## Формат подготовительных артефактов

Результат подготовки контекста складывается в корень проекта.

Общий план выполнения именуется `<TASK-ID>-execution-plan.md`.

Контекстный пакет подзадачи именуется `<TASK-ID>-context-<NN>-<slug>.md`.

Общий план должен содержать исходную задачу, список подзадач, порядок выполнения, зависимости между подзадачами, зоны правок, ссылки на контекстные пакеты, критерии завершения всей задачи и команды проверки.

Каждый контекстный пакет должен содержать цель подзадачи, поведенческий промпт для агента-исполнителя, обязательный read set, краткую выжимку ключевых фактов из прочитанных источников, разрешенные и запрещенные зоны правок, зависимости и входы от других подзадач, ожидаемый результат, проверки, критерии готовности, риски, запреты и открытые вопросы.

Для структуры контекстного пакета используется [templates/context-package-template.md](./templates/context-package-template.md).

Для запуска роли сборщика контекста используется [prompts/context-collector/prompt.md](./prompts/context-collector/prompt.md).
