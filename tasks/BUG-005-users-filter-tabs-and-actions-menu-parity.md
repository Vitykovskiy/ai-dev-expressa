# Карточка задачи

## Карточка задачи

- Идентификатор: `BUG-005`
- Родительская задача: `FEATURE-004`
- Заголовок: `Фильтры и меню действий пользователей не соответствуют прототипу`
- Единица поставки: `FEATURE-004`
- Роль: `Фронтенд`
- Контурная карта: `docs/architecture/application-map/frontend-backoffice.md`
- Приоритет: `Высокий`
- Статус: `Готова к работе`

## Цель

`Исправить воспроизводимые расхождения клиентской части вкладки Пользователи с .references/Expressa_admin: фильтры должны повторять прототип, а меню действий пользователя должно показывать разделитель между группами действий и стабильное расстояние между иконкой и текстом пункта.`

## Детали дефекта

- Контур причины: `frontend`.
- Затронутые сценарии: `F004-SC-002`, `F004-SC-008`.
- Шаги воспроизведения:
  1. Открыть вкладку `Пользователи`.
  2. Сравнить кнопки фильтра `Все`, `Баристы`, `Заблокированные` с `.references/Expressa_admin/src/app/components/FilterTabs.tsx` и `.references/Expressa_admin/src/app/screens/UsersScreen.tsx`.
  3. Открыть меню действий пользователя через кнопку с иконкой `MoreVertical`.
  4. Сравнить пункты меню с `.references/Expressa_admin/src/app/components/UserActionsMenu.tsx`.
- Фактический результат: кнопки фильтра не соответствуют прототипу; в выпадающем меню визуально отсутствует разделитель между группами действий; между иконкой пункта меню и названием пункта нет ожидаемого расстояния.
- Ожидаемый результат: активный фильтр отображается как заполненная pill-кнопка по прототипу, неактивные фильтры отображаются как серые pill-кнопки по прототипу; меню действий сохраняет разделитель между действиями назначения или снятия роли и действием блокировки; каждый пункт меню сохраняет расстояние между иконкой и текстом как в `UserActionsMenu.tsx`.

## Границы задачи

- Входит исправление визуального соответствия фильтров вкладки `Пользователи` прототипу `.references/Expressa_admin`.
- Входит исправление визуального соответствия меню действий пользователя: разделитель между опциями и расстояние между иконкой и названием пункта.
- Входит проверка, что действия `Назначить роль`, `Снять роль баристы`, `Заблокировать` и `Разблокировать` остаются в текущей системной области доступности и не включают операции вне области FEATURE-004.
- Не входит изменение поведения назначения роли в серверной части.
- Не входит изменение блокировки, разблокировки, создания пользователя или снятия роли `barista` как системного поведения.
- Не входит изменение `.references/Expressa_admin`.

## Проверки

- Вручную открыть `Пользователи` и сравнить фильтры `Все`, `Баристы`, `Заблокированные` с `.references/Expressa_admin/src/app/components/FilterTabs.tsx`.
- Вручную открыть меню действий активного пользователя и проверить разделитель между действиями назначения или снятия роли и `Заблокировать`.
- Вручную проверить расстояние между иконкой и текстом для пунктов `Назначить роль`, `Снять роль баристы`, `Заблокировать` и `Разблокировать`, если соответствующий пункт доступен по состоянию пользователя.
- Проверить `F004-SC-002` для поиска и фильтров списка пользователей.
- Проверить `F004-SC-008` для визуального соответствия вкладки `Пользователи` и меню действий.
- `npm run test:frontend`
- `npm run typecheck:frontend`
