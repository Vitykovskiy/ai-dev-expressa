# Сценарий: administrator управляет статусом доступа пользователя

## Граница

Один сценарий: блокировка или разблокировка пользователя на экране `Пользователи`.

## Параметры сценария

- Триггер: administrator нужно изменить статус доступа пользователя.
- Актор: administrator.
- Цель: прекратить или восстановить доступ пользователя к приложению.
- Исход: статус доступа пользователя изменен.

## Основной поток

1. Administrator открывает backoffice через служебного Telegram-бота.
2. Administrator переходит во вкладку `Пользователи`.
3. Administrator выбирает пользователя.
4. Administrator выбирает действие изменения статуса доступа пользователя.
5. Система сохраняет новый статус доступа.
6. Доступ пользователя к приложению обновляется в соответствии с новым статусом.

## Альтернативы и исключения

- Administrator блокирует пользователя, и пользователь теряет доступ к приложению.
- Administrator разблокирует пользователя, и пользователь получает доступ к приложению в соответствии со своими ролями.
- Экран `Пользователи` образует единую бизнес-поверхность управления доступом пользователей и также поддерживает назначение ролей в рамках соседнего сценария управления ролями.
