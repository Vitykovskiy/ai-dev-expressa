# Вариант использования: Customer Create Pickup Order

## Граница

Один вариант использования: создание customer заказа на выдачу на слот текущего дня.

## Источники

- `docs/business/scenarios/customer-place-pickup-order.md`
- `docs/business/business-rules/access-and-roles.md`
- `docs/business/business-rules/menu-catalog-and-options.md`
- `docs/business/business-rules/pickup-slots-and-capacity.md`
- `docs/business/business-rules/order-lifecycle-and-history.md`

## Триггер

Customer открывает customer-приложение через клиентского Telegram-бота и инициирует оформление заказа.

## Предусловия

- Пользователь идентифицирован через клиентского Telegram-бота или test-mode исключение.
- Пользователь не заблокирован.
- Актуальный каталог доступен customer.
- Для выбора доступны только слоты текущего дня, сформированные внутри рабочих часов.

## Основной поток

1. Система допускает customer в customer-приложение только после Telegram-центричного входа или test-mode исключения.
2. Система отображает customer меню, сгруппированное по категориям.
3. Система предоставляет карточку выбранного товара.
4. Если товар является напитком, система требует выбор одного размера из `S`, `M`, `L`.
5. Система принимает количество товара.
6. Система отображает доступные дополнительные опции товара в составе назначенных групп.
7. Если группа опций взаимоисключающая, система допускает выбор не более одной опции из группы.
8. Система добавляет настроенную позицию в корзину.
9. Система предоставляет customer просмотр и редактирование корзины до подтверждения создания заказа.
10. Система отображает доступные слоты выдачи текущего дня.
11. Customer выбирает слот выдачи.
12. Customer инициирует создание заказа.
13. Система валидирует доступность выбранного слота и допустимость состава корзины.
14. Система создаёт заказ из корзины.
15. Система присваивает заказу статус `Создан`.
16. Система учитывает заказ в вместимости выбранного слота.
17. Система делает созданный заказ доступным в истории заказов customer.

## Альтернативы и исключения

- Если пользователь заблокирован, система не допускает его к приложению.
- Если customer пытается открыть приложение напрямую по URL вне Telegram-режима, система не допускает рабочий доступ.
- Если слот недоступен, система не создаёт заказ и требует выбрать другой доступный слот.
- Если customer не выбрал обязательный размер напитка, система не создаёт позицию заказа.
- Если customer выбрал более одной опции из взаимоисключающей группы, система не принимает такой состав.

## Постусловия

- Создан новый заказ в статусе `Создан`.
- Заказ связан с customer и выбранным слотом.
- Вместимость слота уменьшена на единицу доступного остатка.

## Открытые вопросы

- Во входных материалах не зафиксировано, должна ли система резервировать слот на этапе предварительного выбора до фактического создания заказа.
- Не зафиксировано поведение системы при конкурентной попытке нескольких customer занять последнее доступное место в слоте.
