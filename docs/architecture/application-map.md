# Карта приложения

## Назначение

Этот документ является коротким навигационным индексом по исполняемым контурам, общим пакетам, проверкам и маршрутам поставки. Детали реализации, точки входа, зависимости, команды запуска и feature-срезы хранятся в контурных картах.

Разработчик не должен читать весь репозиторий или все контурные карты перед стартом задачи. Рабочий контекст собирается в таком порядке:

1. `README.md`.
2. Родительская `FEATURE-*` и назначенная дочерняя задача.
3. Профильный стандарт из `docs/architecture/` и контурная карта из задачи.
4. Документы из поля `Минимальный read set`.
5. Точечный поиск по коду только внутри путей, указанных задачей и контурной картой.

## Контурные карты

| Контур | Карта | Когда читать |
| --- | --- | --- |
| Серверная часть | `docs/architecture/application-map/server.md` | `BE-*`, серверные части `AR-*`, задачи с изменением HTTP API, доменной логики, guard-правил или слоя хранения данных |
| Внутренний административный контур | `docs/architecture/application-map/backoffice-web.md` | `FE-*`, клиентские части `AR-*`, задачи с изменением Vue-приложения, маршрутов, хранилищ состояния, HTTP-клиентов или UI-поведения |
| Общие типы | `docs/architecture/application-map/shared-types.md` | Задачи, меняющие DTO, причины отказа, роли, вкладки или другие общие контракты между контурами |
| Проверки и поставка | `docs/architecture/application-map/quality-and-delivery.md` | `QA-*`, `DO-*`, задачи с изменением e2e-пути, конвейера, контейнерного маршрута, дымовой проверки или развёртывания |

## Текущие контуры

| Контур | Путь | Статус |
| --- | --- | --- |
| Серверная часть | `apps/server` | Реальный исполняемый контур на `NestJS` для HTTP API, bootstrap доступа во внутренний административный контур, серверных guard-правил и модуля управления каталогом меню |
| Внутренний административный контур | `apps/backoffice-web` | Реальное Vue-приложение на `Vue 3`, `Vite`, `Vuetify`, `Vue Router` и `Vitest` для ролей `administrator` и `barista` |
| Клиентское веб-приложение | `apps/customer-web` | Целевой путь будущего клиентского Telegram веб-приложения; каталог пока не создан |
| Общие типы | `packages/shared-types` | Реальный общий пакет деклараций типов для контрактов между серверной и клиентской частью |
| Сквозные проверки | `e2e/` | Playwright-наборы `FEATURE-*` |
| Инфраструктура и конвейер | `infra/`, `.github/workflows/` | Compose-маршрут, скрипты развёртывания, дымовая проверка и GitHub Actions |

## Правила актуализации

- Обновляйте только те контурные карты, чьи факты изменились.
- Обновляйте этот индекс, если появился новый исполняемый контур, общий пакет, контурная карта или изменилась навигация по картам.
- Обновляйте `docs/architecture/README.md`, если меняется состав обязательных архитектурных артефактов или маршрут поиска.
- Обновляйте корневой `README.md`, если корневая навигация по репозиторию стала неполной.
- Не добавляйте в этот индекс feature-детали, которые нужны только одной реализации. Такие детали должны жить в `FEATURE-*`, дочерних задачах, релевантных `docs/system/` и соответствующей контурной карте.
