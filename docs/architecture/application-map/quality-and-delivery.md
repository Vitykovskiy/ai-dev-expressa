# Карта проверок и поставки

## Назначение

Эта карта фиксирует фактическое устройство сквозных проверок, инфраструктурных артефактов, конвейера и маршрутов развёртывания. Её читают задачи `QA-*`, `DO-*` и задачи, которые меняют путь тестирования, сборки, дымовой проверки или развёртывания.

## Контуры и ответственность

| Контур | Путь | Ответственность | Основные зависимости |
| --- | --- | --- | --- |
| Сквозные проверки | `e2e/` | Playwright-наборы `FEATURE-*`, которые поднимают нужные исполняемые контуры и проверяют законченный feature-сценарий | `@playwright/test`, `apps/server`, `apps/backoffice-web` |
| Infra | `infra/` | Compose-маршрут `FEATURE-001`, шаблоны окружения, VPS-скрипт развёртывания и Node-скрипт дымовой проверки | `Docker`, `Docker Compose`, исполняемые контуры |
| Конвейер | `.github/workflows` | Проверка ветки, сборка, compose-валидация, локальная дымовая проверка и развёртывание `FEATURE-001` на VPS | `GitHub Actions`, `infra/` |

## Текущий состав

- `e2e/feature-001` — Playwright-набор `QA-001`, который поднимает `apps/server` и `apps/backoffice-web` для проверки Telegram-входа `administrator`.
- `e2e/feature-002` — Playwright-набор `QA-002`, который использует те же исполняемые контуры и проверяет сквозное управление каталогом меню `administrator`: категорию, товар с ценой и группу дополнительных опций.
- `infra/docker/compose.feature001.yml` — контейнерный маршрут `FEATURE-001`.
- `infra/docker/.env.example` и `infra/docker/.env.server.example` — шаблоны окружения контейнерного маршрута.
- `infra/scripts/deploy-feature001.sh` — воспроизводимое развёртывание `FEATURE-001` на VPS.
- `infra/scripts/smoke-backoffice-access.mjs` — дымовая проверка доступа во внутренний административный контур.
- `.github/workflows/ci.yml` — модульные проверки, сборка, compose-валидация и локальная дымовая проверка.
- `.github/workflows/deploy-feature001.yml` — развёртывание `staging/production` и post-deploy дымовая проверка.
- Для контейнерного маршрута `FEATURE-001` шаблоны окружения лежат в `infra/docker/.env.example` и `infra/docker/.env.server.example`.

## Feature-срезы

### `FEATURE-001`: доступ во внутренний административный контур

- E2E-набор поднимает серверную и клиентскую часть через Playwright `webServer`.
- Проверяются рабочий вход `administrator`, отказ при прямом рабочем доступе без Telegram и test environment-прогон через `SMOKE_MODE=test`.
- VPS-маршрут синхронизирует репозиторий на сервер, рендерит env-файлы, запускает `infra/scripts/deploy-feature001.sh` и завершает выпуск post-deploy дымовой проверкой.
- Локальная и конвейерная проверка отдельно подтверждают рабочий вход `administrator`, отказ при прямом рабочем доступе без Telegram и выделенный test environment-прогон через `SMOKE_MODE=test`.

### `FEATURE-002`: управление каталогом меню

- Использует существующие команды сборки, модульных тестов и e2e-проверок для `apps/server`, `apps/backoffice-web` и Playwright.
- E2E-набор поднимает серверную и клиентскую часть через Playwright `webServer`, выполняет Telegram-вход `administrator`, сохраняет структурный снимок каталога и повторно читает его из UI после перезагрузки страницы.
- Отдельные команды, env-файлы и workflow для текущей архитектурной рамки не требуются.
- Отдельная задача `DO-*` не выделяется, пока фича укладывается в существующие исполняемые контуры, переменные окружения и маршрут развёртывания `FEATURE-001`.

## Команды запуска и проверки

- Установка зависимостей: `npm install`.
- E2E-проверка: `npm run test:e2e`.
- Проверка контейнерного маршрута `FEATURE-001`: `npm run deploy:feature001:config`.
- Дымовая проверка `FEATURE-001`: `npm run smoke:backoffice-access`.
- Сборка серверной части: `npm run build --workspace @expressa/server`.
- Сборка клиентской части: `npm run build --workspace @expressa/backoffice-web`.
- Модульные и HTTP-тесты серверной части: `npm run test --workspace @expressa/server`.
- Модульные тесты клиентской части: `npm run test --workspace @expressa/backoffice-web`.
- Маршруты развёртывания для конкретных окружений читаются из `deployment-map.md`.

## Когда обновлять карту

- Появился новый e2e-набор или изменился путь тестов.
- Изменился compose-файл, шаблон окружения, скрипт развёртывания или дымовая проверка.
- Изменились workflow, этапы конвейера, условия запуска или post-deploy проверка.
- Изменился маршрут развёртывания, отката или порядок восстановления.
