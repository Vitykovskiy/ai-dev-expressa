# Карточка задачи

## Карточка задачи

- Идентификатор: `FE-010`
- Родительская задача: `FEATURE-002`
- Заголовок: `Базовые токены и общие menu-примитивы для переноса дизайна`
- Описание: `Нужно подготовить в apps/backoffice-web базовый визуальный слой для переноса дизайна menu из прототипа .references/Expressa_admin: токены, карточки, кнопки действий, бейджи, пустые состояния, строки списков и диалоговые обертки, которые можно переиспользовать на всех menu-маршрутах без копирования CSS по страницам. Задача ограничена внутренними примитивами клиентской части и не должна менять публичные HTTP-контракты, DTO, маршруты menu.* или хранилище состояния menuCatalogStore. Оркестрация, UX-валидации и сценарии сохранения остаются в композиционных функциях и хранилищах состояния; новые компоненты отвечают только за отображение и локальные UI-состояния.`
- Единица поставки: `FEATURE-002`
- Роль: `Фронтенд`
- Изменяемый контур: `frontend`
- Приоритет: `Критический`
- Статус: `Готова к работе`

## Примечания

- Зависимости: `Отсутствуют внутри серии FE-010...FE-017; задача опирается на текущую реализацию FEATURE-002 в apps/backoffice-web.`
- Минимальный read set: `docs/architecture/frontend-architecture.md`, `docs/architecture/application-map/backoffice-web.md`, `docs/system/domain-model/menu-catalog.md`, `docs/system/contracts/menu-and-availability-management.md`, `docs/system/use-cases/administrator-manage-menu.md`, `docs/system/ui-behavior-mapping/backoffice-ui-binding.md`
- Ожидаемый результат для ревью: `В apps/backoffice-web появился единый набор menu-примитивов и визуальных токенов, который воспроизводит визуальный язык прототипа и используется как общая база для shell, категорий, товаров и групп дополнительных опций без дублирования стилей по страницам.`
- Проверки: `Модульные тесты новых визуальных примитивов и их локальных состояний, если в них есть нетривиальная логика; локальная проверка сборки и подключения примитивов к menu-экранам; npm run test:backoffice.`
- Обновление карты приложения: `Не требуется, если структура apps/backoffice-web не меняется; при выделении новых общих каталогов, точек входа или маршрутов обновить docs/architecture/application-map/backoffice-web.md и README.md при необходимости.`
- Критерии готовности: `Задача завершена, когда menu-подпоток получил общие токены и переиспользуемые визуальные примитивы, достаточные для последующего переноса shell, списка категорий, списка товаров и редакторов без повторного изобретения локального UI-слоя.`
