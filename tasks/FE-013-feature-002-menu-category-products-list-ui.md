# Карточка задачи

## Карточка задачи

- Идентификатор: `FE-013`
- Родительская задача: `FEATURE-002`
- Заголовок: `Экран списка товаров выбранной категории`
- Описание: `Нужно перенести в apps/backoffice-web экран списка товаров выбранной категории поверх FE-012: заголовок категории, возврат назад, CTA добавления товара, пустое состояние и строки товаров должны собраться в тот же визуальный поток, что и экран категорий. Задача ограничена визуальным слоем и локальными UI-состояниями списка; публичные HTTP-контракты, DTO, маршруты menu.* и menuCatalogStore менять нельзя. Оркестрация навигации, dirty-state и сценариев сохранения должна оставаться в текущих композиционных функциях и хранилищах состояния.`
- Единица поставки: `FEATURE-002`
- Роль: `Фронтенд`
- Изменяемый контур: `frontend`
- Приоритет: `Высокий`
- Статус: `Готова к работе`

## Примечания

- Зависимости: `FE-012`
- Минимальный read set: `docs/architecture/frontend-architecture.md`, `docs/architecture/application-map/backoffice-web.md`, `docs/system/domain-model/menu-catalog.md`, `docs/system/contracts/menu-and-availability-management.md`, `docs/system/use-cases/administrator-manage-menu.md`, `docs/system/ui-behavior-mapping/backoffice-ui-binding.md`
- Ожидаемый результат для ревью: `Экран списка товаров выбранной категории визуально согласован с экраном категорий и shell menu, корректно обрабатывает пустой список, возврат назад и переход к созданию или редактированию товара без изменения существующих маршрутов и сценариев оркестрации.`
- Проверки: `Локальная проверка перехода categories -> products -> back, пустого состояния категории и CTA добавления товара; модульные тесты экранных адаптеров или компонентов списка товаров при наличии ветвящейся UI-логики; npm run test:backoffice.`
- Обновление карты приложения: `Не требуется, если сохраняется текущая структура menu-маршрутов; при изменении состава крупных компонентов списка товаров или навигационных узлов обновить docs/architecture/application-map/backoffice-web.md и README.md при необходимости.`
- Критерии готовности: `Задача завершена, когда список товаров выбранной категории работает в едином визуальном потоке с categories-экраном и подготавливает непротиворечивый вход в карточку товара FE-014.`
