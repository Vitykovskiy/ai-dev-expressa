# Правила: доступ и роли

## Граница

Один набор правил: доступ пользователей к `Expressa v1` и управление ролями в рамках Telegram-центричного продукта.

## Область действия

- Клиентский веб-интерфейс.
- Backoffice.
- Управление ролями, блокировкой и разблокировкой пользователей.

## Правила

| ID     | Условие / контекст                                       | Правило                                                                                                |
| ------ | -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| AR-001 | Пользователь открывает клиентский сценарий               | Доступ customer к продукту предоставляется через клиентского Telegram-бота.                            |
| AR-002 | Пользователь открывает backoffice                        | Доступ barista и administrator к backoffice предоставляется через отдельного служебного Telegram-бота. |
| AR-003 | Пользователь активировал клиентского бота                | Пользователь получает роль `customer`.                                                                 |
| AR-004 | Пользователь заблокирован administrator                  | Заблокированный пользователь не может пользоваться приложением.                                        |
| AR-005 | Administrator разблокировал пользователя                 | Разблокированный пользователь может пользоваться приложением в соответствии со своими ролями.          |
| AR-006 | Продукт введён в эксплуатацию                            | В системе существует один исходный главный administrator как первичный носитель административных прав. |
| AR-007 | Administrator управляет ролями                           | Administrator может назначать пользователей на роль `barista`.                                         |
| AR-008 | Главный administrator управляет ролями                   | Главный administrator может назначать пользователей на роль `administrator`.                           |
| AR-009 | Administrator управляет доступом пользователей           | Administrator может блокировать пользователей.                                                         |
| AR-010 | Administrator управляет доступом пользователей           | Administrator может разблокировать пользователей.                                                      |
| AR-011 | Пользователь работает в backoffice                       | Доступ к функциям backoffice определяется ролью пользователя.                                          |
| AR-012 | Пользователь обращается к веб-интерфейсу напрямую по URL | Прямой доступ без прохождения через Telegram в рабочем режиме v1 не поддерживается.                    |

## Исключения

- Для тестирования допускается сценарий без Telegram-авторизации; это исключение ограничено тестовым режимом и не меняет бизнес-правило обязательности Telegram для v1.
