# Вариант использования: Barista Confirm Order

## Граница

Один вариант использования: подтверждение barista заказа, ожидающего обработки.

## Источники

- `docs/business/scenarios/barista-confirm-order.md`
- `docs/business/business-rules/access-and-roles.md`
- `docs/business/business-rules/backoffice-operations.md`
- `docs/business/business-rules/order-lifecycle-and-history.md`

## Триггер

Barista выбирает заказ из очереди и инициирует подтверждение.

## Предусловия

- Пользователь идентифицирован через служебного Telegram-бота.
- Пользователь имеет роль `barista` или роль с доступом к вкладке `Заказы`.
- Заказ находится в статусе `Создан`.

## Основной поток

1. Система допускает barista в backoffice.
2. Система предоставляет очередь заказов и детали выбранного заказа, включая состав и итоговую сумму.
3. Barista инициирует подтверждение заказа.
4. Система проверяет допустимость перехода `Создан` -> `Подтвержден`.
5. Система переводит заказ в статус `Подтвержден`.
6. Система сохраняет идентификатор barista, подтвердившего заказ.
7. Система отправляет customer уведомление об изменении статуса.

## Альтернативы и исключения

- Если заказ уже не находится в статусе `Создан`, система не выполняет подтверждение.

## Постусловия

- Заказ имеет статус `Подтвержден`.
- Audit по подтверждению сохранён.
