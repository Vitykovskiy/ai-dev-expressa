# Use Case: Barista Receive Order Reminder

## Граница

Один use case: отправка системой напоминания barista о заказах, ожидающих действий.

## Источники

- `docs/business/scenarios/barista-receive-order-reminder.md`
- `docs/business/business-rules/backoffice-operations.md`
- `Expressa — Требования к продукту.txt`

## Триггер

В системе существуют заказы, ожидающие действий barista, и наступает очередной момент проверки необходимости напоминания.

## Предусловия

- Существуют заказы, ожидающие действий barista.
- Для barista доступен Telegram backoffice-бот как канал напоминаний.

## Основной поток

1. Система выявляет наличие заказов, ожидающих действий barista.
2. Система формирует содержимое Telegram-напоминания.
3. Система отправляет напоминание через Telegram backoffice-бота.
4. Barista получает напоминание и может вернуться к обработке заказов в backoffice.

## Альтернативы и исключения

- Если заказы, ожидающие действий, отсутствуют, система не отправляет напоминание.

## Постусловия

- Barista получил Telegram-напоминание о заказах, ожидающих действий.

## Открытые вопросы

- Периодичность отправки напоминаний во входных материалах не определена.
- Во входных материалах не определено, должно ли напоминание направляться всем barista или конкретному подмножеству.
