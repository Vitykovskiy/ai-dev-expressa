# Карта развёртывания

## Контуры и окружения

| Окружение | Назначение | Обязательные проверки |
| --- | --- | --- |
| `local` | Разработка и ручная проверка дочерних задач | локальный запуск нужных контуров, модульные тесты, локальный smoke-сценарий |
| `ci` | Проверка запроса на слияние и готовности к слиянию | установка зависимостей, lint, модульные тесты, сборка |
| `staging` | Проверка интеграции перед выпуском на VPS | развёртывание через GitHub Actions, дымовая проверка критического сценария, запуск существующих e2e по затронутым фичам, проверка окружения и конфигурации |
| `production` | Боевой выпуск | контролируемое развёртывание, post-deploy smoke-check, путь отката |

## CI/CD-поток

- `pipeline запроса на слияние`
  - установка зависимостей
  - run lint
  - запуск модульных тестов по затронутым контурам
  - сборка
- `main/release pipeline`
  - запуск в `GitHub Actions`
  - сборка исполняемых артефактов
  - публикация контейнерных образов
  - развёртывание на VPS в целевое окружение
  - запуск существующих e2e для затронутых фич
  - execute smoke-check

## Базовые переменные окружения

- `ADMIN_TELEGRAM_ID`
- `DISABLE_TG_AUTH`
- `DATABASE_URL`
- `TG_CUSTOMER_BOT_TOKEN`
- `TG_BACKOFFICE_BOT_TOKEN`

Каждая новая переменная окружения должна быть описана здесь вместе с контуром, где она используется.

## Smoke-check и откат

- Каждое развёртывание должно иметь зафиксированный smoke-сценарий для затронутой `FEATURE-*`.
- E2e-тесты создаются и поддерживаются в `QA-*`; DevOps-задача обеспечивает только их запуск в требуемом окружении.
- Каждое развёртывание должно иметь документированный откат или порядок восстановления.
- Если изменение затрагивает migration или несовместимый контракт, стратегия отката должна быть описана до начала реализации.

## Правила актуализации

- Обновляйте документ при изменении этапов pipeline, целей развёртывания, переменных окружения, работы с секретами, smoke-check или процесса отката.
- Если из этого документа уже нельзя восстановить путь от merge до проверенного развёртывания, документ устарел и должен быть обновлен в той же задаче.
