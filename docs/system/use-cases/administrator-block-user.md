# Вариант использования: Administrator Block User

## Граница

Один вариант использования: блокировка пользователя administrator.

## Источники

- `docs/business/scenarios/administrator-block-user.md`
- `docs/business/business-rules/access-and-roles.md`
- `docs/business/business-rules/backoffice-operations.md`

## Триггер

Administrator инициирует блокировку пользователя.

## Предусловия

- Пользователь идентифицирован через служебного Telegram-бота.
- Пользователь имеет роль `administrator` и доступ к вкладке `Пользователи`.

## Основной поток

1. Система предоставляет administrator список пользователей.
2. Administrator выбирает пользователя.
3. Administrator инициирует блокировку.
4. Система сохраняет признак блокировки пользователя.
5. Система прекращает доступ пользователя к приложению.

## Альтернативы и исключения

- Во входных материалах не определено отдельное поведение для уже заблокированного пользователя при повторной блокировке.

## Постусловия

- Пользователь имеет статус blocked и не может пользоваться приложением.
