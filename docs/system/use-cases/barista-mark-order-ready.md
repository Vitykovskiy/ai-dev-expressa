# Вариант использования: Barista Mark Order Ready

## Граница

Один вариант использования: перевод подтверждённого заказа в состояние `Готов к выдаче`.

## Источники

- `docs/business/scenarios/barista-mark-order-ready.md`
- `docs/business/business-rules/backoffice-operations.md`
- `docs/business/business-rules/order-lifecycle-and-history.md`

## Триггер

Barista инициирует перевод заказа в `Готов к выдаче`.

## Предусловия

- Пользователь идентифицирован через служебного Telegram-бота.
- Пользователь имеет роль `barista` или роль с доступом к вкладке `Заказы`.
- Заказ находится в статусе `Подтвержден`.

## Основной поток

1. Система предоставляет barista подтверждённый заказ.
2. Barista инициирует перевод в `Готов к выдаче`.
3. Система проверяет допустимость перехода `Подтвержден` -> `Готов к выдаче`.
4. Система переводит заказ в статус `Готов к выдаче`.
5. Система сохраняет идентификатор barista, выполнившего перевод.
6. Система отправляет customer уведомление о готовности заказа.

## Альтернативы и исключения

- Если заказ не находится в статусе `Подтвержден`, система не выполняет перевод.

## Постусловия

- Заказ имеет статус `Готов к выдаче`.
- Audit по переводу в `Готов к выдаче` сохранён.
