# Сценарий: administrator управляет ролями пользователей

## Граница

Один сценарий: назначение пользователю роли `barista` или `administrator`.

## Параметры сценария

- Триггер: administrator нужно выдать пользователю операционные права.
- Актор: administrator.
- Цель: предоставить пользователю нужную роль для работы в backoffice.
- Исход: роль пользователя изменена.

## Основной поток

1. Administrator открывает backoffice через Telegram backoffice-бота.
2. Administrator переходит во вкладку `Пользователи`.
3. Administrator выбирает пользователя.
4. Administrator назначает пользователю роль `barista` или `administrator`.
5. Система сохраняет новое ролевое назначение.
6. Доступ пользователя к вкладкам backoffice обновляется в соответствии с новой ролью.

## Альтернативы и исключения

- Исходные требования не уточняют, ограничено ли назначение новых `administrator` только главным administrator.
