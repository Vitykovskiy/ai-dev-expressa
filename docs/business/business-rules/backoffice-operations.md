# Правила: операционная работа backoffice

## Граница

Один набор правил: действия barista и administrator в backoffice при ежедневной работе.

## Область действия

- Обработка заказов в backoffice.
- Временное управление доступностью меню.
- Напоминания barista.
- Аудит действий barista.
- Разделение вкладок backoffice по ролям.
- Управление доступом пользователей на экране `Пользователи`.

## Правила

| ID     | Условие / контекст                                | Правило                                                                                                                                    |
| ------ | ------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| BO-001 | Barista работает с входящими заказами             | Barista видит входящие заказы, их состав и итоговую сумму.                                                                                 |
| BO-002 | Barista управляет доступностью меню               | Barista может временно менять доступность позиций меню и дополнительных опций.                                                             |
| BO-003 | Barista управляет доступностью меню               | Barista не может изменять цены и структуру меню.                                                                                           |
| BO-004 | Administrator управляет меню                      | Administrator может управлять структурой меню, товарами и ценами.                                                                          |
| BO-005 | Administrator управляет операционными параметрами | Administrator может управлять рабочими часами и вместимостью слотов.                                                                       |
| BO-006 | Administrator управляет пользователями            | Administrator может блокировать пользователей.                                                                                             |
| BO-007 | Administrator управляет пользователями            | Administrator может разблокировать пользователей.                                                                                          |
| BO-008 | Administrator работает на экране `Пользователи`   | Экран `Пользователи` объединяет назначение ролей, блокировку и разблокировку как единый бизнес-сценарий управления доступом пользователей. |
| BO-009 | Заказы ожидают действий                           | Barista получает периодические Telegram-напоминания о заказах, ожидающих действий.                                                         |
| BO-010 | Barista подтвердил заказ                          | Система фиксирует, какой barista подтвердил заказ.                                                                                         |
| BO-011 | Barista перевёл заказ в статус `Готов к выдаче`   | Система фиксирует, какой barista выполнил это действие.                                                                                    |
| BO-012 | Barista отклонил заказ                            | Система фиксирует, какой barista выполнил отклонение.                                                                                      |
| BO-013 | Пользователь открывает backoffice                 | Backoffice разделён на вкладки, и доступ к вкладкам определяется ролью пользователя.                                                       |
| BO-014 | Пользователь имеет роль `barista`                 | Barista имеет доступ к вкладкам `Заказы` и `Доступность`.                                                                                  |
| BO-015 | Пользователь имеет роль `administrator`           | Administrator имеет доступ к вкладкам `Заказы`, `Доступность`, `Меню`, `Пользователи` и `Настройки`.                                       |

## Исключения

- Точная периодичность напоминаний barista не определена в исходных требованиях.
